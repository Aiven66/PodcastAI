# DESIGN.md

## 产品与用户画像
- **产品定位**：AI驱动的播客生成平台，面向海外C端用户
- **品牌名**：PodcastAI
- **核心价值**：链接/文件/文本一键转播客，声音克隆，单人/双人播客模式
- **用户群体**：内容创作者、播客爱好者、音频工作者、个人博主

## 气质与意象
- 清晨阳光穿过极简工作室的落地窗，现代设备静静运转，空气中浮动着创意的氛围
- 科技感但不冰冷，像专业录音室的洁净与高效
- 简洁的交互流程，每一步都有明确的引导

## 视觉策略
- 极简主义 + 科技感融合
- 大面积留白，聚焦核心功能
- 微妙的动效增强交互反馈
- 音频波形、声音可视化元素作为视觉点缀

## 配色方案
- **主色**：Teal/青绿色系 - 科技感与清新感结合，区别于常见的蓝紫色AI风格
- **背景**：浅灰白/纯白 - 干净专业
- **辅助色**：深灰色文字、金色accent（付费/高亮）
- **禁止**：蓝紫色渐变、AI味过重的配色

## 字体排版
- **英文主字体**：Inter (Google Fonts) - 清晰易读的UI字体
- **中文备用**：Noto Sans SC - 与Inter风格匹配
- **标题字重**：600-700
- **正文字重**：400-500
- **行高**：1.5-1.6

## 动效与交互
- 页面切换：fade + slide
- 按钮交互：scale微动 + color渐变
- 音频播放：波形动画
- 生成进度：streaming打字机效果
- 缓动曲线：cubic-bezier(0.4, 0, 0.2, 1)

## 页面结构
- **首页**：Hero区（核心功能入口）→ 功能演示 → 价格预览 → 用户反馈
- **控制台**：播客列表 → 生成入口 → 历史记录
- **定价页**：三列对比卡片（免费/基础/Pro）
- **博客**：文章列表 → 文章详情
- **管理后台**：博客管理 → 用户管理

## 响应式
- 移动端优先适配
- 导航栏：桌面端横向 → 移动端汉堡菜单
- 功能区：桌面端居中大卡片 → 移动端堆叠
- 表格：桌面端完整 → 移动端卡片化

## 设计禁忌
- 禁止蓝紫色渐变
- 禁止复杂的多层阴影
- 禁止过重的装饰元素
- 禁止模糊的AI背景图
- 禁止过小的点击区域（移动端最小44px）
- 禁止自动播放音频（需用户触发）
